## 2. 设计原则(本方案遵守的 6 条铁律)

1. **破坏性重构许可**:不向后兼容 a9202cb 之前的任何接口。`from_random` 改 `from_pretrained`,签名变化自由。
2. **架构模仿 diffusers**:每个 `ModelMixin` 子类有 `from_pretrained/save_pretrained`;Pipeline 持多个子模块;Scheduler 独立。
3. **不重写已有架构**:UNet/DiT/VAE/MotionModule/HiFiGAN 都不重写,只补权重接入。
4. **GPU 不是必须**:保持 `device="cpu"` 默认 + 真 offload 钩子。CPU demo 跑通是大模型 e2e 最低门槛。
5. **测试不能掉**:每加一个 `from_pretrained` 都要补 1 个 test。
6. **CI 守卫必保留**:placeholder 扫描器 / hardcoding / degrade_logging 三个 CI 守卫原样保留。

## 5. v0.9.0 详细方案(第 5-8 周,ComfyUI 范式)

### 5.1 DAG 拓扑执行引擎

**当前**:`PipelineBuilder` 走线性顺序。

**目标**:支持 `Pipeline.node("a").node("b", inputs={"x": "a.output"}).node("c", inputs={"x": "b.output"})` 形式的 DAG。

```python
# core/graph.py (新文件,500 行)
class GraphExecutor:
    """拓扑执行 + 指纹缓存 + 增量重算。
    
    参考 ComfyUI CacheSet:
    - 输入指纹:hash(args + kwargs + parent_outputs) → 命中缓存则跳过
    - outputs:节点返回值
    - objects:节点实例(model, pipeline, ...)
    """
    def __init__(
        self,
        nodes: list[NodeSpec],
        edges: list[Edge],
    ): ...
    
    def topo_sort(self) -> list[str]: ...
    
    def execute(
        self,
        *,
        changed_nodes: set[str] | None = None,  # 仅重算这些节点及下游
        cache_key: str = "",
    ) -> dict[str, Any]: ...
```

### 5.2 输入指纹 + LRU 缓存

```python

import hashlib, json

def compute_fingerprint(
    node_id: str, args: dict, kwargs: dict,
    parent_outputs: dict[str, Any],
) -> str:
    """SHA-256(args + kwargs + parent_fingerprints + node_id)。
    
    返回 64-char hex string。
    """
    ...

class HierarchicalCache:
    """outputs 缓存 + objects 缓存,分别 LRU。
    
    参考 ComfyUI CacheSet:
    - outputs[node_id] → node 的返回值(可能几 GB tensor)
    - objects[node_id] → node 持有的模型/资源(可共享)
    """
    def __init__(self, max_outputs_bytes: int = 8*2**30): ...
    def get(self, key: str) -> Any | None: ...
    def set(self, key: str, value: Any) -> None: ...
```

## 11. 总结

### 11.1 一句话总结本方案

> **torcha-verse v0.8.x 的核心是 3 个补强 + 2 个范式转移**:
> - 补强:真权重 / 真 prompt 编码 / 真 sampler 循环
> - 转移:从"echo backend + 随机权重"到"ModelMixin + from_pretrained"
> - 转移:从"线性 Pipeline"到"DAG 拓扑 + 指纹缓存"

### 11.2 借鉴来源汇总

| 范式 | 来源 | 落地 |
|---|---|---|
| DAG 拓扑 + 指纹缓存 | ComfyUI | v0.9.0 |
| ModelPatcher 权重补丁 | ComfyUI | v0.8.5 |
| Sampler.from_pretrained | HunyuanDiT/HunyuanVideo | v0.8.0 |
| predict(prompt, H, W, steps, cfg, flow_shift) | Hunyuan | v0.8.0 |
| ModelMixin + from_pretrained | diffusers | v0.8.0 |
| enable_*_cpu_offload | diffusers | v0.8.5 |
| Schedulers 注册表 | diffusers | v0.9.0 |
| transform_checkpoint_dict_key | diffusers | v0.8.0 |
| extra_model_paths.yaml | ComfyUI | v1.0.0 |

### 11.3 关键决策

1. **不依赖 `transformers` / `diffusers` 包**:保持核心零依赖,自研 BPE / safetensors 解析
2. **接受 CPU 跑得慢**:v0.8.0 阶段允许 SD-Turbo CPU 跑 60s 出图
3. **保持现有节点名不变**:39 节点 spec 不破坏,v0.9.0 引入 PipelineV2 走新引擎
4. **真权重为王**:每个 `ModelMixin` 必须有 `from_pretrained` 测试,缺则 CI 红
